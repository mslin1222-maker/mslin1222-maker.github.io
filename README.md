# Copilot Tool Decision Guide

This project is an interactive guide for **Microsoft 365 Copilot, Copilot Cowork, and Microsoft Scout**. It helps users choose the right Copilot tool for different work tasks. It also explains how Copilot Cowork Credits may be planned, estimated, and used in business scenarios.

The site is published with GitHub Pages. It uses static HTML pages to provide simple navigation, decision guides, comparison tables, scenario examples, and budget planning content.

## Project Goal

As Copilot tools become more powerful, users and managers may ask questions such as:

- Should this task use M365 Copilot or Copilot Cowork?
- Which tasks are worth using Copilot Credits?
- How can we estimate the cost of Copilot Cowork?
- What are the roles of M365 Copilot, Copilot Cowork, and Microsoft Scout?
- How should users, managers, IT teams, and finance teams plan adoption and governance?

This website turns these questions into clear and practical decision tools. It helps users make better choices faster.

## Key Features

### 1. Task Decision Guide

This guide helps users decide which tool is best for a task:

- Use **M365 Copilot** for quick Q&A, summaries, drafts, and single-document tasks.
- Use **Copilot Cowork** for multi-step work across apps, data sources, and files.
- Use **Microsoft Scout** for ongoing tracking, proactive reminders, and background progress.

### 2. Copilot Cowork Credit Playbook

This section explains the basic ideas of Copilot Cowork Credit planning, including:

- Credit cost factors
- Task levels: light, medium, and heavy
- Interactive Credit budget estimator
- Common business scenarios
- Cost-saving tips and governance ideas

### 3. Three-Tool Scenario Guide

This section explains the roles of the three tools:

- **M365 Copilot**: ask and answer, with fast support
- **Copilot Cowork**: assign and complete, with end-to-end delivery
- **Microsoft Scout**: monitor and act, with background support

It also includes:

- A quick decision helper
- A six-point comparison table
- Common work scenarios
- Simple rules of thumb for choosing tools

### 4. Useful for Managers and Adoption Teams

This project is useful not only for individual users, but also for teams that plan or manage Copilot adoption, such as:

- Microsoft 365 Copilot users
- IT administrators
- AI adoption leads
- Finance and procurement teams
- Department managers
- Digital transformation teams

## Content Design Idea

This guide uses a simple routing idea: choose the right tool for the right type of work.

- **M365 Copilot**: best for fast answers, summaries, drafts, single files, and single-step tasks.
- **Copilot Cowork**: best for multi-step work across apps, data sources, and output files.
- **Microsoft Scout**: best for ongoing tracking, proactive reminders, and background progress.

Simple rule:

> Ask Copilot for answers. Assign Cowork for delivery. Let Scout monitor progress.

# Copilot 工具決策指南

一套以 **Microsoft 365 Copilot、Copilot Cowork 與 Microsoft Scout** 為核心的互動式決策指南，協助使用者判斷不同工作情境下，應該使用哪一種 Copilot 工具，並進一步理解 Copilot Cowork 的 Credit 成本結構、預算估算方式與應用場景。

本專案以 GitHub Pages 部署，透過多個靜態 HTML 頁面提供互動式導覽、判斷精靈、對照表、場景庫與成本估算內容。

## 專案目的

在 Copilot 工具逐漸從「問答輔助」走向「端到端交辦」與「背景自主推進」的情境下，許多使用者與管理者會遇到以下問題：

- 這件事該用 M365 Copilot，還是 Copilot Cowork？
- 什麼樣的任務值得消耗 Copilot Credits？
- Copilot Cowork 的 Credit 成本應該如何估算？
- M365 Copilot、Copilot Cowork、Microsoft Scout 三者的分工是什麼？
- 一般使用者、管理者、IT 或財務單位該如何規劃導入與治理？

本網站將這些問題整理成可視化、互動式的決策工具，協助使用者快速做出更適合的選擇。

## 功能亮點

### 1. 任務決策指南

協助使用者快速判斷：

- 單次問答、摘要、草稿與單一文件產出，適合使用 M365 Copilot
- 跨應用、多步驟、多文件產出，適合交給 Copilot Cowork
- 需要持續追蹤、主動提醒與背景推進的工作，適合 Microsoft Scout

### 2. Copilot Cowork Credit 完全攻略

整理 Copilot Cowork 的 Credit 規劃概念，包含：

- Credit 計價因素
- 任務量級：輕量、中等、重度
- 互動式 Credit 預算估算器
- 常見部門應用場景
- 省點心法與成本治理建議

### 3. 三工具場景決策指南

以互動方式說明：

- M365 Copilot：你問我答，快速回應
- Copilot Cowork：你交我辦，端到端完成工作
- Microsoft Scout：不問自來，背景自主推進

並提供：

- 快速判斷精靈
- 六面向對照表
- 常見工作場景
- 一句話心法與組合技建議

### 4. 適合管理者與導入團隊使用

本專案不只適合一般使用者查詢，也適合以下角色作為導入與教育素材：

- Microsoft 365 Copilot 使用者
- IT 管理員
- AI 導入負責人
- 財務與採購規劃人員
- 部門主管
- 數位轉型與生產力推動團隊

## 內容設計理念

本指南採用「分流」的概念，協助使用者把不同任務交給最合適的工具：

- **M365 Copilot**：適合快速問答、摘要、草稿、單一文件或單次任務
- **Copilot Cowork**：適合跨應用、多步驟、多資料來源與多份交付成果
- **Microsoft Scout**：適合需要持續追蹤、主動提醒、背景推進的任務

核心心法：

> 問答找 Copilot，交辦找 Cowork，盯場找 Scout。
